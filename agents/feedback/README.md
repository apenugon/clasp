# Task Feedback

After `SH-014` is complete, every newly integrated task is expected to leave a feedback artifact here.

Each file is named `<TASK-ID>.json` and captures practical feedback from the agent that implemented the task:

- what felt good or bad in the language/tooling
- what future agents should watch for
- concrete follow-up improvements
- the files/tests involved in the task

This directory is meant to help future agents incorporate hard-won task experience into later implementation work instead of rediscovering the same friction repeatedly.
